# Contributing

Thanks for contributing to `wave-driven-dev`.

## Scope

This repository contains reusable skills and templates for Wave-Driven Development workflows.

## How to Contribute

1. Fork the repo and create a branch from `main`.
2. Make focused changes with clear commit messages.
3. Update docs/templates when behavior changes.
4. Open a pull request using the PR template.

## Pull Request Expectations

- Keep changes scoped and explain the reason.
- Include before/after behavior in the PR description.
- Keep skill instructions consistent across:
  - `SKILL.md`
  - `templates/`
  - `README.md` (if user-facing behavior changed)

## Versioning

Version tags and releases are automated by GitHub Actions.
Do not manually edit tags; submit normal PRs and let the release workflow handle version bumps.

## Questions

Open a GitHub issue for discussion before large structural changes.
